# VisionCast-AI Application Icons

This directory contains the source icon assets for VisionCast-AI.

## Files

| File | Description |
|------|-------------|
| `visioncast.svg` | Master vector icon (broadcast/AI theme, 256×256 viewBox) |
| `visioncast.rc`  | Windows resource file — embeds `visioncast.ico` into the `.exe` |
| `visioncast.ico` | Windows multi-resolution icon *(generate from PNG — see below)* |
| `visioncast.icns`| macOS icon bundle *(generate with `iconutil` — see below)* |

> **Note:** `visioncast.ico` and `visioncast.icns` are binary files that must be
> generated by the developer and are **not** committed to the repository.

---

## Generating `visioncast.ico` (Windows)

1. Export `visioncast.svg` to a high-resolution PNG (e.g. 512×512) using Inkscape
   or any SVG renderer:
   ```bash
   inkscape --export-type=png --export-width=512 --export-filename=visioncast_512.png visioncast.svg
   ```

2. Use **ImageMagick** to create a multi-size `.ico` containing 16, 32, 48, 64,
   128, and 256 px layers:
   ```bash
   convert visioncast_512.png \
       \( -clone 0 -resize 256x256 \) \
       \( -clone 0 -resize 128x128 \) \
       \( -clone 0 -resize 64x64  \) \
       \( -clone 0 -resize 48x48  \) \
       \( -clone 0 -resize 32x32  \) \
       \( -clone 0 -resize 16x16  \) \
       -delete 0 visioncast.ico
   ```

   Alternatively, use an online converter such as <https://convertico.com> or
   <https://icoconvert.com>.

3. Place the generated `visioncast.ico` in this directory (`packaging/icons/`).

---

## Generating `visioncast.icns` (macOS)

1. Export the SVG to individual PNG sizes required by macOS:
   ```bash
   mkdir visioncast.iconset
   for size in 16 32 64 128 256 512; do
       inkscape --export-type=png --export-width=$size \
           --export-filename=visioncast.iconset/icon_${size}x${size}.png \
           visioncast.svg
   done
   # Retina (@2x) variants
   for size in 16 32 64 128 256; do
       double=$((size * 2))
       inkscape --export-type=png --export-width=$double \
           --export-filename=visioncast.iconset/icon_${size}x${size}@2x.png \
           visioncast.svg
   done
   ```

2. Bundle into an `.icns` file using **iconutil** (macOS only):
   ```bash
   iconutil -c icns visioncast.iconset -o visioncast.icns
   ```

3. Place the generated `visioncast.icns` in this directory (`packaging/icons/`).

## Using the Icons in the Build

### Qt (cross-platform) — `ui/resources/visioncast.qrc`
The SVG is embedded at compile time via the Qt resource system. `main.cpp` loads
it and calls `QApplication::setWindowIcon()`.

### Windows — CMake / RC file
Add `packaging/icons/visioncast.rc` to the `visioncast_ui` sources in
`ui/CMakeLists.txt` so MSVC/MinGW embeds the icon in the executable.

### macOS — CMake bundle
Set `MACOSX_BUNDLE_ICON_FILE` and add `visioncast.icns` as a resource in
`ui/CMakeLists.txt`.
